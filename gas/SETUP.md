# GAS セットアップ手順書
## Green Carbon HR Tools — ビザ管理 バックエンド設定

このドキュメントに従って設定すると、複数の担当者が同じデータを共有できるようになります。  
所要時間: **約15分**

---

## 前提条件

- Google アカウント（会社の Google Workspace 推奨）
- 管理者権限は不要

---

## Step 1: Google スプレッドシートを作成

1. [Google スプレッドシート](https://sheets.google.com) を開く
2. 「空白」をクリックして新規作成
3. タイトルを **「Green Carbon - ビザ管理データ」** に変更（わかりやすい名前なら何でも可）

---

## Step 2: Apps Script を開く

1. メニューから **「拡張機能」→「Apps Script」** をクリック
2. 新しいタブで Apps Script エディタが開く
3. タイトルを **「visa_manager_backend」** に変更

---

## Step 3: コードを貼り付ける

1. エディタ左側の `コード.gs` をクリック
2. 既存のコード（`function myFunction() {...}`）を**全部削除**
3. このリポジトリの `gas/visa_manager.gs` の内容を**全てコピーして貼り付け**
4. **Ctrl+S**（Mac: Cmd+S）で保存

---

## Step 4: 初回実行（権限の許可）

1. 上部のメニューから関数を **`initSheets`** に変更
2. ▶️ 実行 ボタンをクリック
3. 「権限を確認」→「詳細」→「visa_manager_backend（安全でないページ）に移動」→「許可」
4. スプレッドシートに戻ると **VisaRecords / DocStatuses / Memos** の3シートが作成される

---

## Step 5: ウェブアプリとしてデプロイ

1. Apps Script エディタ右上の **「デプロイ」→「新しいデプロイ」** をクリック
2. 歯車アイコン → **「ウェブアプリ」** を選択
3. 以下の通りに設定：

   | 項目 | 設定値 |
   |------|--------|
   | 説明 | `visa_manager v1` |
   | 次のユーザーとして実行 | **自分（your@email.com）** |
   | アクセスできるユーザー | **全員** |

4. **「デプロイ」** をクリック
5. 表示された **ウェブアプリURL** をコピー（`https://script.google.com/macros/s/...`）

> ⚠️ このURLが外部からデータにアクセスするためのキーです。社外に漏らさないよう管理してください。

---

## Step 6: HTMLにURLを設定

1. リポジトリの `tools/visa_manager.html` を開く
2. 以下の行を見つける（9行目付近）：

   ```javascript
   const GAS_URL = 'https://script.google.com/macros/s/AKfycbw...（現在のURL）';
   ```

3. コピーした新しいURLに書き換える：

   ```javascript
   const GAS_URL = 'https://script.google.com/macros/s/（コピーしたURL）/exec';
   ```

4. Git commit & push → Vercel に自動デプロイ

---

## Step 7: 動作確認

1. `https://green-carbon-hr-tools.vercel.app/tools/visa_manager.html` にアクセス
2. ヘッダーのバッジが **「● GAS接続中」**（緑）になっていれば成功
3. 社員を追加→スプレッドシートに行が追加されることを確認

---

## CSVインポートで98名を一括登録する方法

1. ビザ管理画面の **「📥 CSVインポート」** ボタンをクリック
2. **「テンプレートをダウンロード」** でCSVを取得
3. テンプレートに従って社員データを入力（Excelで編集可）
4. 入力済みCSVをドラッグ&ドロップ または ファイル選択でインポート
5. プレビューを確認して **「取り込む」** をクリック
6. GASに接続済みなら自動的にスプレッドシートにも書き込まれる

## トラブルシューティング

**「● サンプルデータ」のまま変わらない**
→ デプロイURLが正しく設定されているか確認。`/exec` で終わっているかチェック。

**「エラー」バッジが表示される**
→ GASのデプロイ設定で「アクセスできるユーザー: 全員」になっているか確認。「再デプロイ」を試す。

**スプレッドシートにデータが書き込まれない**
→ `testConnection` 関数をApps Scriptから手動実行してエラーを確認。

---

## 更新時（コードを変更した場合）

GASのコードを変更したら必ず「新しいデプロイ」ではなく **「デプロイを管理」→「編集（鉛筆アイコン）」→「バージョン: 新しいバージョン」→「デプロイ」** を使う。  
URLは変わらないため、HTMLの書き換えは不要。
